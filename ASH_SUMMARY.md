## What Is ASH?

ASH is an ownerless, mineable ERC-20 on **Bittensor EVM** (chainId 964).

The only way new ASH can exist is for someone to burn compute — solve keccak-256 puzzles, submit proof-of-work shares on-chain, and claim their pro-rata cut of each epoch pool.

- **No admin key. No premine. No upgrades.** Contract is frozen at deploy.
- **21 million hard cap.** 50 ASH/epoch, halving every 210,000 epochs (~4 years).

## The Mining Process

Each 10-minute **epoch**:

1. Contract publishes a `seed` (chained from prior epoch's share count — not precomputable)
2. Miners grind: `keccak(seed ‖ miner_address ‖ nonce) < target`
3. Valid nonces are submitted on-chain as **shares**
4. Epoch flips → each miner claims `my_shares / total_shares × 50 ASH`

### Why address-binding matters

The nonce includes the miner's address in the preimage. A nonce found by miner A is **worthless** to miner B — the contract verifies the submitter matches. This prevents mempool frontrunning, a real exploit in naive PoW tokens.

## How It Differs From Bitcoin

| | Bitcoin | ASH |
|---|---|---|
| Hash function | SHA-256 | keccak-256 |
| Winner model | One winner per block | **Pro-rata pool** |
| Reward | Winner takes all | Proportional to shares submitted |
| Nonce security | Anyone can steal from mempool | Address-bound — theft-proof |
| Settlement | On block | Claim after each epoch |
| Difficulty | Retargets every 2016 blocks | Bang-bang ×2/÷2 per epoch |
| Admin | Protocol rules | **None — frozen contract** |

Pro-rata pooling means small miners always earn something proportional to their work. No luck required.

## The Edge

**1. Narrative fits Bittensor perfectly**
ASH is explicitly positioned as *proof that someone burned decentralized GPU compute* (Lium SN51, Targon SN4). Every unit is a receipt for destroyed energy routed through Bittensor's compute subnets. This isn't bolted on — it's the core premise.

**2. Ownerless = trustless**
Nobody can change the rules, pause the contract, or take fees. This is rare. Most "fair launch" tokens have a multisig somewhere. ASH has zero admin surface.

**3. Agent-native**
The onboarding is designed for AI agents, not humans. Any agent can be given ONBOARDING.md and autonomously set up wallets, get funded, and start mining. This is a distribution moat — humans onboard miners one by one; agents can onboard in parallel at scale.

**4. Solo mining window**
Right now there is zero competition. One miner earns 100% of every epoch. Early miners accumulate ASH at cost — the only cost is gas (~0.002 TAO/epoch).

## What's Needed Next

**Priority 1 — Distribution**
Get other agents mining. This is the main focus. The onboarding flow exists:
- `https://github.com/OSHA-B/ash-v1/blob/main/ONBOARDING.md`
- Raw (paste into agent): `https://raw.githubusercontent.com/OSHA-B/ash-v1/main/ONBOARDING.md`

Each new miner needs:
1. The miner scripts (`git clone https://github.com/OSHA-B/ash-v1.git`)
2. An EVM keypair (generated by `ash.py provision`)
3. ~0.5 TAO for gas (funded once, lasts ~30 days)

**Priority 2 — GPU kernel**
When competition arrives and difficulty rises, CPU loses. A CUDA keccak kernel replacing the Python loop in `pod_miner.py` would give ~10,000× hashrate on an A100. Until then, CPU is sufficient.

**Priority 3 — Domain**
Panel currently at long ICP canister URL. `ashpod.io` (~$32/yr) would make sharing easier. Deferred until there are users.

## Economics

- **Gas:** ~0.002 TAO/epoch → 0.5 TAO lasts ~250 epochs (~35 days at 7/day)
- **CPU cost:** ~$0.00/epoch (local machine)
- **Lium A100:** ~$0.07–$0.30/hr, ~10,000× CPU hashrate
- **At genesis (solo):** 50 ASH/epoch × 7/day = 350 ASH/day, no competition

Value of ASH is currently zero — it's a fresh token. Value accrues if/when miners compete for it, exchanges list it, or utility is built on top.
